The changes to the source code are minimal and a working binaray can be built using
`cmake` or `make` as usual.

== Building with automake

To build using `automake`, run `dev/scripts/automake/build.sh`.
The install directory is `$WORKDIR/bin/games/Maelstrom/`.

== Building with cmake

Works from the command line and using the Microsoft extensions from "CMake Tools".

== Building documentaion with doxygen

----
cd $WORKDIR/dev/doxygen
rm -rf docs/html docs/latex
doxygen
----

Run `doxygen -x` to check the difference with default `Doxyfile`. 

== Building documentation with Sphinx

First build the `doxygen` documentation as above, then run:
----
cd $WORKDIR/dev/sphinx
make clean
make html
----
